コーディングもターミナルもバイブコーディングも初めての人のための、Claude Code / Codex CLI スキルコレクション。

「これ作って」と言っても、すぐにコードを書き始めない — **まず聞く。** 何を作りたいのか、隠れた決定事項は何か、本当に必要なものは何か。整理が終わって初めて作り始める。

## なぜ必要か

Claude CodeやCodexはすでに強力だが、初めての人には3つ足りないものがある：

1. **隠れた決定を先に引き出す** — 「ログイン機能を作って」と言った時、メールログインかソーシャルログインか、パスワードリセットは必要か — 自分でも気づいていなかった決定を代わりに聞いてくれる。
2. **専門用語を初回だけ説明** — 隔離（元のファイルに触れない別のコピー）、コミット（作業の保存ポイント）のように、初めて出てきた時だけ括弧で説明し、以降は繰り返さない。
3. **セッションをまたぐ平易な記憶** — 「前回どこまでやったっけ？」と聞けば、技術ログではなく「ログインページのデザインまで決めた。次は会員登録フロー」と教えてくれる。

この3つ以外は、Claude Code / Codexのネイティブ機能（権限確認、巻き戻し、サンドボックス）を**置き換えず、その上に載せる。**

## どう動くか

話しかければ自動的に適切なステップに案内する：

```
話しかける → 一緒に整理 → 隔離 → 小さな計画 → 一歩ずつ作る → 確認 → 記憶
```

| ステップ | スキル | 役割 |
|----------|--------|------|
| 入口 | `senpai` | 常にオン — リクエストを聞いて適切なスキルに接続 |
| 整理 | `senpai-brainstorming` | 一つずつ質問して隠れた決定を引き出す。コードは一行も書かない |
| 隔離 | `senpai-isolate` | プロジェクトのコピーを作ってそこで作業。元のファイルには絶対触れない |
| 計画 | `senpai-plan` | 合意した設計を2〜5分の小さなステップに分割。承認まで何も作らない |
| 作成 | `senpai-build` | 計画を一歩ずつ実行。各ステップ後にレビュー。エラーはここで修正 |
| 確認 | `senpai-finish` | 実際に動かして確認し、結果を残すか捨てるか聞く |
| 記憶 | `senpai-remember` | 何をしたか平易な言葉で保存。次回そのまま続けられる |

## セーフティ

スキルはガイドラインだが、**コードで強制される**ものが一つある — `PreToolUse`ガードフック：

- **秘密ファイルをブロック** — `.env`、SSH鍵、`.pem`/`.key`、名前に「secret」や「credential」を含むファイル。読み取り・書き込み・シェル参照すべてブロック。
- **設定ファイルを保護** — `.claude/`、`.codex/`、`.claude-plugin/`内のファイルはセッション中に変更不可。

隔離ワークスペース内でも同様に適用される。*"This looks like a secret file"*が表示されたら、バグではなくガードが正常に動作している証拠。

残りの安全モデル — 承認するまで本物のプロジェクトに触れないこと — は`senpai-isolate`（隔離）と`senpai-finish`（最終決定）が担当する。

## インストール

**Claude Code：**

```
/plugin marketplace add calmtiger86/senpai-ask
/plugin install senpai-ask@senpai-ask
```

**Codex CLI：** このリポジトリをCodexプラグインとしてインストール（Codexのプラグインドキュメント参照 — 標準Claude Code / Codex互換スキルプラグイン、特別な設定不要）。

インストール後は普通に話しかけるだけ — 作りたいものを言えば`senpai`が始める。
